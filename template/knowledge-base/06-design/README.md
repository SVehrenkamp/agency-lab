# 06 · Design

Run a brand/style interview first (irreducibly human — this is your taste), then
produce a design system, then the comprehensive designs that account for every UX
flow.

- **Consumes:** `04-scoping/v1-scope.md` + `03-refinement/prd.md` + all of `05-ux/`
- **Produces:** `brand-brief.md`, `design-system.html`, `design.md`, `designs.html`
- **Default mode:** `mode:collab` (taste) + auto-draft (production)
- **Backing skills:** `design:design-system`, `design:design-critique`
- **Design source:** self-contained HTML built in the repo with Claude — no Figma. `design-system.html` is the visual render and `design.md` the canonical written style guide (the version downstream phases cite); `designs.html` composes the system into screens for every UX flow.
